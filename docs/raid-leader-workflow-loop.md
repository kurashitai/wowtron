# Raid Leader Workflow Loop

## Goal

Keep WoWtron useful for `next pull` and `next raid night`, not only for one isolated log screen.

## Core Outputs

Every boss analysis should be able to produce:

- `Pull Brief`
- `Session Command Brief`
- `Night Recap`
- `Guild Boss Knowledge`
- `Coaching Memory`

## Quality Questions

When reviewing the RL workflow, ask:

- Can an RL find the blocker in under 30 seconds?
- Can an officer copy a useful recap without rewriting it?
- Does the night comparison sound believable?
- Does the recap tell the team what to keep, what to change, and what to watch?
- Is the start call for the next night actually usable?

## Improvement Cadence

Weekly:

- review exported `Session Brief` and `Night Recap` examples
- flag wording that sounds too generic or too analytical

Biweekly:

- compare session recap against real officer notes
- mark missing blockers, wrong priorities, and noisy coaching calls

Monthly:

- promote wording and recap formats that RLs/officers reuse without editing
- remove sections that do not change decisions

## Product Standard

The RL workflow is good when:

- the user does not need to open five more tabs to understand the wipe
- the next night can start from the recap itself
- guild memory gets stronger over time instead of resetting every session
